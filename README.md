# Проект: Взаимодействие с LLM через API

Этот проект демонстрирует работу с языковыми моделями через API сервиса GigaChat.

## Общая настройка

### Установка зависимостей

Убедитесь, что установлены все необходимые зависимости:
```bash
pip install gigachat
```

### Настройка конфигурации

Настройте конфигурацию в файле `config.py`, указав следующие параметры:
- `GIGACHAT_API_KEY` — ваш API-ключ
- `GIGACHAT_SCOPE` — необходимая область доступа
- `GIGACHAT_MODEL` — модель, которую вы хотите использовать
- `CERTIFICATE_PATH` — путь к сертификату (если требуется)

## День 1: Первый запрос к LLM через API

Этот скрипт демонстрирует отправку запроса к модели языкового процессора (LLM) через API сервиса GigaChat и вывод ответа в консоль.

### Функционал

- Отправка пользовательского запроса к модели GigaChat
- Получение и вывод ответа модели
- Отображение статистики использования токенов:
  - Количество токенов в запросе (prompt tokens)
  - Количество сгенерированных токенов (completion tokens)
  - Общее количество использованных токенов
  - Количество кэшированных токенов

### Использование

1. Запустите скрипт:
   ```bash
   python day1.py
   ```

2. Введите ваш запрос в консоль и получите ответ от модели.

## Пример работы

```
Введите запрос для LLM GigaChat:    Расскажи о себе

Привет! Я — языковая модель, разработанная для помощи в ответах на вопросы, создании текстов и решении различных задач.

Prompt tokens: 15, completion token: 25, total tokens: 40, cached tokens: 0
```

## День 2: Контроль формата ответа

Этот скрипт демонстрирует различные способы контроля ответа от LLM через API GigaChat.

### Функционал

- Сравнение ответов на один и тот же запрос:
  - Без ограничений
  - С явным описанием формата ответа
  - С ограничением по длине (максимальное количество токенов)
  - С условием завершения ответа (stop sequence)
- Использование system prompt для управления структурой ответа
- Тестирование различных методов ограничения вывода модели

### Использование

1. Убедитесь, что конфигурация в `config.py` уже настроена

2. Запустите скрипт:
   ```bash
   python day2.py
   ```

3. Программа автоматически выведет и сравнит четыре различных типа ответов.

### Пример работы

```
Ответ по умолчанию:
-------------------
Астрономия — это наука о Вселенной, её объектах и явлениях...

Ответ в соответствии с описанным форматом:
------------------------------------------

Астрономия необходима для понимания структуры Вселенной, происхождения и эволюции звёзд, планет и других космических объектов.

Ограниченный по длине ответ:
----------------------------
Астрономия — это наука о

Условие завершения ответа:
--------------------------
Астрономия — это наука о Вселенной
```

## День 3: Разные способы рассуждения

Этот скрипт демонстрирует различные подходы к решению одной и той же логической задачи с использованием LLM через API GigaChat.

### Функционал

- Решение задачи четырьмя разными способами:
  - Прямой ответ без дополнительных инструкций
  - Пошаговое решение с явной инструкцией
  - Решение через сгенерированный промпт
  - Решение от группы экспертов с разными ролями (аналитик, физик, школьник)
- Сравнение точности и качества различных подходов
- Анализ влияния стратегии рассуждения на результат

### Использование

1. Убедитесь, что конфигурация в `config.py` уже настроена

2. Запустите скрипт:
   ```bash
   python day3.py
   ```

3. Программа выведет решения задачи всеми четырьмя методами для сравнения.

### Пример задачи

> Рыцари всегда говорят только правду. Лжецы всегда говорят только ложь. На острове рыцарей и лжецов собралась компания из 12 человек. Каждый человек заявил всем остальным: "Вы все лжецы!". Сколько лжецов может быть в этой компании?

## День 4: Влияние температуры на ответы LLM

Этот скрипт демонстрирует влияние параметра температуры на ответы языковой модели через API GigaChat.

### Функционал

- Выполнение одного и того же запроса с разными значениями температуры:

- Анализ применимости различных значений температуры для разных типов задач

### Использование

2. Запустите скрипт:
   ```bash
   python day4.py
   ```

3. Программа выведет ответы для всех трех значений температуры для сравнения.

### Пример запроса

> Расскажи в двух предложениях о том, как можно человеку попасть на луну?

## День 6: Первый агент с контекстом

Этот скрипт реализует простого агента для взаимодействия с LLM, способного поддерживать контекст диалога.

### Функционал

- Принятие пользовательских запросов через CLI-интерфейс
- Инкапсуляция логики взаимодействия с LLM в отдельной сущности (классе SimpleAgent)
- Сохранение и передача контекста диалога между запросами
- Форматированный вывод ответов с использованием Markdown
- Обработка ошибок при обращении к API

### Использование

1. Убедитесь, что конфигурация в `config.py` уже настроена

2. Запустите скрипт:
   ```bash
   python day6.py
   ```

3. Введите свои запросы по очереди

4. Для завершения работы агента введите команду "выход", "exit" или "quit"

### Особенности

Агент сохраняет полную историю диалога, включая:
- Все пользовательские запросы (роль USER)
- Все ответы агента (роль ASSISTANT)

При каждом новом запросе полный контекст передается в LLM, что позволяет агенту учитывать предыдущие сообщения при генерации ответов.

## Требования

- Python 3.8+
- Библиотека `gigachat`
- Библиотека `rich` (для форматированного вывода)
- Действующий доступ к API GigaChat
